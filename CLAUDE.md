# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
yarn start       # Dev server with hot reload (localhost:3000)
yarn build       # Generate static site to build/
yarn serve       # Serve the built site locally
yarn clear       # Clear Docusaurus cache
yarn deploy      # Deploy to GitHub Pages (gh-pages branch)
```

Dependencies are managed with yarn (`yarn` to install).

## Architecture

This is a **Docusaurus 3.9.2** static documentation site for algorithm/data structure interview preparation, deployed to GitHub Pages at `https://pdimeglio-dev.github.io/interview-cheat-sheet/`.

**Stack:** Docusaurus + React 19 + MDX, Node 20+ required.

**CI/CD:** `.github/workflows/deploy.yml` auto-builds and deploys on push to `main` using GitHub Actions.

## Content

All interview content lives in `/docs/` as markdown files. The sidebar is auto-generated from the directory structure (`sidebars.js` uses `autogenerated` type) — no manual sidebar entries needed when adding new docs.

- New topic pages go in `/docs/` as `.md` or `.mdx` files
- Frontmatter (`title`, `description`) is optional but improves navigation
- `onBrokenLinks` is set to `'warn'` (not error), so broken internal links won't fail the build

Site config is in `docusaurus.config.js`. Custom CSS is in `src/css/custom.css`. Static assets go in `static/`.
